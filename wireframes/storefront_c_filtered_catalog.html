Option C: Filtered Catalog Storefront — Design decisions & implementation notes:
  • Three-tier navigation: utility bar (account/help links), main nav (logo, category-scoped search, cart), category bar (product types). Maps cleanly to existing _welcome_header.html.erb structure with product_types helper populating the category nav.
  • Prominent search with category dropdown — posts to products#create (existing search action). Category dropdown populated from ProductType.all.
  • Left sidebar filters for category, price, brand, rating, availability. Filter changes use Turbo Frames to reload only the product grid, not the whole page. Filters map to existing ProductFilters concern on Product model.
  • 4-column product grid with rich info: name, star rating, price (with sale/discount), shipping status, variant count. All data available from existing Product, Variant, and Brand models.
  • Pagination using will_paginate (already in the project).

    • Product Detail Page

  • Large image gallery on left, details panel on right. Below: tabbed sections for Description (from product.description), Properties (from product.product_properties), and Reviews. Related products carousel at bottom.
  • Variant selector shows individual variant prices, stock status (variant.sold_out?, variant.low_stock?), and updates dynamically via Stimulus.

    • Cart Page

  • Full page with modernized table layout (no inline styles). Product images, variant details, quantity selectors, line totals. Right sidebar: order summary with subtotal, shipping estimate, coupon code input (shopping/coupons#create), and checkout button.

    • Checkout

  • Multi-step flow matching existing controllers: addresses → shipping methods → payment → confirmation. Each step in a Turbo Frame with progress indicator. Order summary sidebar persists across steps.

    • My Account

  • Left sidebar navigation (matching the filter sidebar pattern) with links to all myaccount sections. Content area on right loads via Turbo Frames.

    • Tech Stack

  • Custom CSS, no framework dependency. Information-dense layout using CSS Grid.
  • Turbo Drive for all navigations. Turbo Frames for filters, cart updates, checkout steps, account sections.
  • Stimulus for: search autocomplete, filter toggle/collapse, sort, view toggle (grid/list), variant selection, quantity stepper.
  • All existing controllers and routes unchanged.
