Option A: Minimal / Editorial Storefront — Design decisions & implementation notes:
  • Sticky minimal nav with logo left, category links, centered search, cart/account right. Maps directly to existing _welcome_header.html.erb partial. Product type categories populate the nav links via product_types helper.
  • Slide-out cart drawer replaces dedicated cart page for quick interactions. Uses a Stimulus controller to toggle open/closed. Cart contents loaded via Turbo Frame from shopping/cart_items.
  • Full-bleed hero with overlaid text — homepage (welcome#index) features a single large hero. The featured product (Product.featured) drives the content.
  • 3-column product grid with hover quick-add. Product listing (products#index) uses this grid. Quick-add posts to shopping/cart_items#create via Turbo and updates the cart drawer via Turbo Stream.
  • Category strip links to products_path(product_type_id: pt.id) for each ProductType.

    • Product Detail Page

  • Split layout: large image gallery (left, from product.images) + details panel (right) with variant selector (product.active_variants), price range display, and "Add to Cart" button.
  • Variant selection uses a Stimulus controller to update price and stock status without page reload.

    • Checkout Flow

  • Keeps existing multi-step flow (addresses → shipping → payment → confirmation) but in a clean single-column centered layout with progress indicator at top.
  • Each step is a Turbo Frame that replaces only the form area, keeping the progress bar and order summary sidebar intact.

    • My Account

  • Clean tabbed interface replacing the current sidebar. Maps to existing myaccount/* controllers (orders, addresses, credit_cards, overview, referrals, store_credits).

    • Tech Stack

  • Replaces Foundation with custom CSS using CSS custom properties — no framework dependency. Propshaft-compatible.
  • Turbo Drive for all page navigations (zero-reload feel).
  • Turbo Frames for cart drawer, quick-add, variant selection.
  • Stimulus for cart toggle, search, nav dropdowns, image gallery.
  • All existing routes and controllers remain unchanged — this is purely layout and view layer.
