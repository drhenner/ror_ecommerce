Option D: Mobile-First Card-Based Storefront — Design decisions & implementation notes:
  • Mobile-first responsive design with actual CSS @media queries built in. Product grid shifts from 4 columns (desktop) to 2 columns (mobile). Resize this wireframe to see the responsive behavior.
  • Bottom tab bar on mobile (Home, Search, Categories, Wishlist, Cart) provides thumb-friendly navigation. Hidden on desktop. A Stimulus controller manages active state and swaps content.
  • Swipeable category pills at top — horizontal scrollable bar populated from ProductType.all. Uses native overflow-x scrolling (no JS needed), or a Stimulus controller for snap behavior.
  • Card-based product UI with rounded corners, soft shadows, hover lift animation, and reveal-on-hover wishlist button. Each card links to products#show.
  • Horizontal scroll "Quick Shop" row for recently viewed or popular items — native touch scrolling works perfectly on mobile.
  • Collection cards with gradient backgrounds act as visual entry points to filtered product lists (products_path(product_type_id: ...)).

    • Product Detail Page

  • Full-width swipeable image carousel (touch-native on mobile). Sticky bottom "Add to Cart" bar appears on scroll with product name, price, and CTA button. Variant selection as tappable chips.
  • On mobile, the add-to-cart bar sits above the bottom tab bar. On desktop, it's inline.

    • Cart

  • Mobile: bottom sheet that slides up from the cart tab, showing card-style items with swipe-to-remove.
  • Desktop: slide-out drawer from right (similar to Option A) or full page with card layout.

    • Checkout

  • Single-page checkout on mobile with accordion sections (address, shipping, payment). On desktop, two-column layout with form left and order summary right.

    • My Account

  • Grid of icon cards on mobile (Orders, Addresses, Cards, Profile, Referrals, Credit) that link to individual pages. On desktop, standard tab layout.

    • Tech Stack

  • Custom CSS with CSS custom properties + real @media breakpoints. No framework dependency.
  • Turbo Drive for SPA-like page transitions.
  • Turbo Frames for: product grid infinite scroll (append more cards), cart drawer, variant selection, checkout accordion sections.
  • Stimulus for: bottom tab bar, category pill selection, image carousel, sticky add-to-cart, wishlist toggle, cart drawer/sheet.
  • All existing controllers and routes unchanged.
