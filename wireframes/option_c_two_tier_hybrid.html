Option C: Two-Tier Hybrid Navigation — Key design decisions:
  • Two-tier navigation — dark primary bar for top-level sections (Orders, Merchandise, Inventory, etc.) + white contextual section bar that swaps sub-links based on the active section. The section bar is itself a <turbo-frame> that updates when you click a primary link.
  • Master-detail pattern — the fulfillment view shows an order list on the left and detail on the right. Clicking an order loads the detail pane via a Turbo Frame, no full navigation. Perfect for high-volume order processing.
  • Turbo Streams for timeline — the order timeline updates in real-time via Action Cable + Turbo Streams. When a shipment is created, the timeline item appears instantly for all admins viewing that order.
  • Contextual actions — "Create Shipment" button in the detail header submits via Turbo Stream, updating both the order status badge and timeline simultaneously.
  • 12-column CSS Grid — a utility-free grid system using plain CSS. No framework dependency, responsive-ready, easily themed.
  • Chart area — would replace the current Morris.js/Raphael charts with a Stimulus controller wrapping Chart.js or a lightweight alternative. Period selector buttons swap chart data via Turbo Frames.
  • This layout best suits high-throughput operations — the two-tier nav keeps context (you're always in "Orders") while the master-detail avoids constant back-and-forth navigation.
